JavaScript is an interpreted client side scripting language that means it runs at client-side inside internet browsers. The running way of javascript is interesting. As it is an interpreted language, not compiled, so it needs to be compiled before run. There is a program called compiler that translates javascript codes into bytecode that the browser javascript engine understands and can execute. Different browsers have different javascript engines to run it such as Chrome has V8, Firefox has spidermonkey, Edge has Chakra, etc.

JavaScript is a single threaded language. It executes all operations on a single thread but with a smart data structure it gives us the execution with multi-threading. The JS event loop is responsible for that process. It keeps some operations in a list called call stack to be executed and some operations in the list called event queue to be executed after call stack functions. Whenever a function from the call stack is executed, it's removed from the stack. Event loop constantly checks whether the call stack list is empty or not. If it's empty, all functions from the event queue are sent to the call stack to be executed maintaining the correct sequence of queue data structure and finally the event queue gets empty as well when all functions are sent to the stack.

The localStorage and sessionStorage are same as each other. Just the difference is that data in sessionStorage is cleared when the page sesion ends with page reload or close, while data in localSorage doesn't clear. Opening a new tab or window creates a new page session that lasts as long as the tab or window is open. While localStorage lasts always even if the tab, window, or browser is closed.
